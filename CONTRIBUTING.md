# Contributing to Harness Engineering Guide

Thank you for your interest in contributing! This guide is a community effort and we welcome all forms of contribution.

## Ways to Contribute

### 📝 Content
- **Guide sections** — Expand existing topics or add new ones
- **Tutorials** — Write step-by-step tutorials with code examples
- **Tool reviews** — Add reviews of harness tools and frameworks
- **Paper summaries** — Summarize relevant academic papers

### 📰 Weekly Digest
- Submit interesting links, papers, or tools you've found
- Open an issue with the tag `weekly-submission`

### 🌐 Translations
- We plan to support Chinese (zh), Japanese (jp), Korean (kr), and more
- Create a new directory (e.g., `zh-guide/`) mirroring the English structure

### 🐛 Fixes
- Typos, broken links, outdated information — all welcome

## How to Submit

1. Fork the repository
2. Create a branch: `git checkout -b your-topic`
3. Make your changes
4. Submit a Pull Request with a clear description

## Writing Style

- **Be practical** — Include code examples and real-world scenarios
- **Be objective** — When comparing tools, present trade-offs fairly
- **Be concise** — Respect the reader's time
- **Cite sources** — Link to original papers, blog posts, and repositories

## Code of Conduct

Be respectful, constructive, and inclusive. We're all here to learn.
